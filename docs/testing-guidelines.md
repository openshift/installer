# Testing Guidelines

## Running Tests

| Command | Scope | Environment |
|---------|-------|-------------|
| `hack/go-test.sh` | Unit tests (`./cmd/... ./data/... ./pkg/...`) with `-short` flag | Podman container (golang:1.25) |
| `go test ./pkg/...` | Unit tests directly | Local Go toolchain |
| `hack/go-integration-test.sh` | Integration tests (`-run .Integration`) | Local; requires `oc`, `nmstatectl`, registry credentials |
| `hack/go-integration-test-nodejoiner.sh` | Node joiner integration tests | Local; auto-installs `setup-envtest` for kubebuilder binaries |

Integration tests are gated by `testing.Short()` -- they are skipped when `hack/go-test.sh` passes `-short`, and run only by the integration scripts.

## Test File Organization

- Test files live alongside the code they test, using the standard `_test.go` suffix.
- Platform validation tests: `pkg/types/<platform>/validation/platform_test.go` and `machinepool_test.go`.
- Platform install-config validation tests (with cloud API mocking): `pkg/asset/installconfig/<platform>/validation_test.go`.
- Agent manifest tests share helpers in `pkg/asset/agent/manifests/util_test.go`.
- Integration test scripts and fixtures: `cmd/openshift-install/testdata/` and `cmd/node-joiner/testdata/`.

## Assertions Library

Use `github.com/stretchr/testify/assert` (not `require`). The codebase does not use `testify/suite`.

```go
assert.Regexp(t, expectedErrMsg, errs)
assert.Empty(t, errs)
assert.NoError(t, err)
```

For validation tests that produce `field.ErrorList`, convert with `.ToAggregate()` before asserting.

## Table-Driven Tests

Nearly all tests use table-driven style. The standard shape is:

```go
func TestValidatePlatform(t *testing.T) {
    cases := []struct {
        name     string
        platform *aws.Platform
        expected string // regex matched against error
    }{
        {name: "minimal", platform: &aws.Platform{Region: "us-east-1"}},
        {name: "invalid region", platform: &aws.Platform{Region: ""}, expected: `...`},
    }
    for _, tc := range cases {
        t.Run(tc.name, func(t *testing.T) {
            err := ValidatePlatform(tc.platform, field.NewPath("test-path")).ToAggregate()
            if tc.expected == "" {
                assert.NoError(t, err)
            } else {
                assert.Regexp(t, tc.expected, err)
            }
        })
    }
}
```

Use `field.NewPath("test-path")` as the base path in validation unit tests.

## Cloud Client Mocking (gomock)

Cloud API clients are defined as interfaces (e.g., `pkg/asset/installconfig/gcp/client.go` defines `API`) with `//go:generate mockgen` directives. Generated mocks live in sibling `mock/` directories with `_generated.go` suffix.

### Regenerating Mocks

```sh
hack/go-genmock.sh
```

This runs `go generate` on `./pkg/asset/installconfig/...` and `./pkg/destroy/...`. It uses `go.uber.org/mock/mockgen`.

**Note**: The `pkg/asset/mock/filefetcher_generated.go` mock is not regenerated by this script. To regenerate it, run:

```sh
go generate ./pkg/asset/filefetcher.go
```

### Mock locations (non-vendor)

| Package | Mock directory |
|---------|---------------|
| `pkg/asset/installconfig/aws/` | `mock/awsroute53_generated.go` |
| `pkg/asset/installconfig/azure/` | `mock/azureclient_generated.go` |

### Usage Pattern in Tests

```go
mockCtrl := gomock.NewController(t)
defer mockCtrl.Finish()
gcpClient := mock.NewMockAPI(mockCtrl)

// Set up expectations with AnyTimes() for reuse across subtests
gcpClient.EXPECT().GetNetwork(gomock.Any(), validNetworkName, validProjectName).
    Return(&compute.Network{}, nil).AnyTimes()
gcpClient.EXPECT().GetNetwork(gomock.Any(), gomock.Not(validNetworkName), gomock.Any()).
    Return(nil, fmt.Errorf("404")).AnyTimes()
```

Use `gomock.Not()` to set up catch-all expectations for invalid inputs alongside specific valid-input expectations.

## FileFetcher Mock Pattern

Asset tests that load YAML from disk use `mock.MockFileFetcher`:

```go
mockCtrl := gomock.NewController(t)
fileFetcher := mock.NewMockFileFetcher(mockCtrl)
fileFetcher.EXPECT().FetchByName("install-config.yaml").
    Return(&asset.File{Filename: "install-config.yaml", Data: []byte(tc.data)}, nil)
```

## Edit Functions Pattern (Cloud Validation Tests)

`pkg/asset/installconfig/{gcp,azure,ibmcloud,powervs}/validation_test.go` use composable edit functions to modify a base valid config per test case:

```go
type editFunctions []func(ic *types.InstallConfig)

invalidateRegion = func(ic *types.InstallConfig) { ic.GCP.Region = "bad" }

// In test cases:
{name: "bad region", edits: editFunctions{invalidateRegion}, expectedError: true}

// In test runner:
ic := validInstallConfig()
for _, edit := range tc.edits {
    edit(ic)
}
```

## Valid Config Factories

Each test file provides a `validInstallConfig()`, `validPlatform()`, or `getValidOptionalInstallConfig()` function that returns a minimal correct object. Tests mutate a single field to produce one specific error. Follow this pattern -- do not build invalid configs from scratch.

## Shared Test Helpers

- `pkg/asset/agent/manifests/util_test.go` -- factories for `OptionalInstallConfig`, `AgentConfig`, `AgentClusterInstall`, NMState configs, and raw YAML fixtures used across agent manifest tests.
- `internal/tshelpers/custom_commands.go` -- custom testscript commands (`isoCmp`, `isoIgnitionContains`, etc.) for ISO content verification in integration tests.
- `internal/tshelpers/fakeregistry.go` -- fake container registry for integration tests.

## Integration Tests (testscript)

Agent and image-based integration tests use `github.com/rogpeppe/go-internal/testscript`. Test cases are `.txt` or `.txtar` files under `cmd/openshift-install/testdata/`.

### testscript file structure

```text
# Comment describing the test
exec openshift-install agent create image --dir $WORK
stderr 'expected output regex'
exists $WORK/agent.x86_64.iso

-- install-config.yaml --
apiVersion: v1
baseDomain: test.metalkube.org
...

-- agent-config.yaml --
apiVersion: v1alpha1
...
```

### Key conventions

- `TestMain` registers the binary under test via `testscript.RunMain` so `exec openshift-install` works.
- Test directories are discovered automatically by `runAllIntegrationTests()` walking `testdata/` subtrees.
- Custom commands (e.g., `isoCmp`, `ignitionImgContains`) are registered in the `Cmds` map.
- `e.Cd` is set to the `data/` directory so embedded resources load correctly.
- `OPENSHIFT_INSTALL_RELEASE_IMAGE_OVERRIDE` and `AUTH_FILE` env vars are forwarded from CI.

### Node Joiner envtest Integration

`cmd/node-joiner/nodejoiner_integration_test.go` uses `sigs.k8s.io/controller-runtime/pkg/envtest` to spin up a local kube-apiserver and etcd. The `hack/go-integration-test-nodejoiner.sh` script installs the required binaries via `setup-envtest`. Do not run these tests with `go test` directly.

## HTTP Mocking

Some tests use `github.com/jarcoal/httpmock` to intercept HTTP calls (e.g., GCP service endpoint validation). Activate per-test:

```go
httpmock.Activate()
defer httpmock.DeactivateAndReset()
httpmock.RegisterResponder("HEAD", url, handler)
```

## Validation Error Matching

Validation functions return `field.ErrorList`. Test `expected` strings are regexes matched with `assert.Regexp`. Escape field path dots and brackets:

```go
expected: `^test-path\.serviceEndpoints\[0\]\.url: Invalid value: "(.*)": host cannot be empty`
```

When no error is expected, leave `expected` empty (or `expectedError: false`) and assert `NoError`/`Empty`.
